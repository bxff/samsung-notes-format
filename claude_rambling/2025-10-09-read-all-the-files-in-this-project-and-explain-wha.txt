⏺ This project is a Samsung Notes (sdocx) to TXT converter written in Python. Here's what it does in detail:

  Project Overview

  The sdocx-to-txt-python-converter extracts text content from Samsung Notes files (.sdocx format) and converts them to plain text format.

  Core Functionality

  Main script (sdocxToTxt.py):
  1. File Preparation: Creates a zippedFiles directory and copies .sdocx files from sdocxFiles folder, renaming them with .zip extension
  2. Extraction: Extracts the note.note file from each renamed zip archive (these are the actual note contents)
  3. Content Processing: Processes extracted note files to clean up formatting and consolidate them
  4. Output Generation: Creates a consolidated allNotes.txt file with processed content

  Supporting modules:
  - fileToList.py: Reads note files using cp1252 encoding and converts them to lists of lines
  - delSpaces.py: Cleans up text by removing consecutive whitespace and non-printable characters
  - listToFile.py: Writes processed content to the output file with formatting (90-character lines with word wrapping)

  Technical Process

  1. Samsung Notes files are actually ZIP archives containing a note.note file with the actual content
  2. The script leverages this by renaming .sdocx to .zip and extracting the internal files
  3. Text processing removes excessive spacing and formats the output for readability
  4. All notes are consolidated into a single output file

  File Structure

  - sdocxFiles/ - Input directory for Samsung Notes files
  - zippedFiles/ - Intermediate directory containing renamed zip files
  - notePnote/ - Directory for extracted .note files
  - allNotes.txt - Final consolidated output file

  The project provides a simple way to extract and consolidate text content from Samsung Notes for backup, analysis, or migration
  purposes.

  Samsung Notes (.sdocx) Format

  Container Structure:
  - .sdocx files are essentially ZIP archives containing the note data
  - The actual text content is stored in a file called note.note inside the archive
  - This is similar to how Microsoft Office files are structured (as ZIP containers)

  Internal note.note Format:
  Based on the extracted content I can see in allNotes.txt, the note.note file contains:

  1. Binary Header/Metadata: The note starts with binary data including:
    - UUID identifiers (like 99a66d6e-46c4-11f0-95ef-433493b81192)
    - Pen/stroke information (com.samsung.android.sdk.pen.pen.preload.FountainPen)
    - Other device-specific metadata
  2. Text Content: Mixed within the binary data are the actual text portions:
    - Readable text like "Mako OT N problem still exists"
    - Formatting characters and control codes
    - Whitespace and newline characters
  3. Encoding: The notes use cp1252 (Windows-1252) encoding, which is a character encoding for Western European languages

  Format Characteristics:
  - Proprietary binary format with embedded text
  - Contains both stroke/pen data (for handwritten content) and typed text
  - Not a simple plain text or standard document format
  - Requires special extraction to access the readable content

  This explains why the converter needs to:
  1. Extract from the ZIP container
  2. Handle the binary format carefully
  3. Filter out non-printable characters and pen metadata
  4. Clean up spacing to extract just the readable text portions

  The format is designed for Samsung's Notes app to preserve both handwritten and typed content with all the original formatting and
  device-specific metadata.
